### モジュールと性能仕様

このプロジェクトは、性能目標の異なる2つの独立したコンポーネントで構成されています。

#### パート1: 履歴バッチ処理 (`data_ingestion/` と `cuda_compute_engine/`)

これは中核となるスループットエンジンであり、ハードウェアの物理限界で巨大な履歴データセットをパースし計算するために設計されています。以下のベンチマークは**このモジュールのみに**適用されます。

![NVIDIA Nsight Profiling](README.assets/nsight-profile.png.png)
*(CPU の `pread`、PCIe Host-to-Device DMA、GPU Kernel 実行が完全にオーバーラップしている)*

**対象ハードウェア（この構成向けにハードチューニング済み）:**
* **CPU:** Intel Core i7-13650HX（14コア、20スレッド）
* **GPU:** NVIDIA GeForce RTX 4060 Laptop GPU（8GB GDDR6）
* **RAM:** 16GB DDR5 4800MHz（厳しい物理制約により Out-of-Core アーキテクチャを強制）
* **ストレージ:** PCIe 3.0 NVMe SSD

**実行指標:**
* **データセット:** Binance BTC/USDT Spot の履歴約定データ 139 GB。パーサはこの特定の CSV 形式にハードコードされており、ETH や他のシンボル/データ型に使うと**高確率でパースに失敗します**。
* **実行時間:** **約60秒**（実際にストップウォッチで計測）。
* **持続スループット:** **約2.31 GB/s**。
> *性能に関する注記: 安定して60秒で動作する版は、16GB RAM 制約向けに調整されています。さらに過激な実験版では **約3.23 GB/s** のピークスループット（139GB を43秒、同様にストップウォッチ計測）に達しましたが、積極的なメモリ事前確保戦略により、限られた16GB物理RAMを圧迫してシステムフリーズを引き起こす可能性があります。*

#### パート2: リアルタイムネットワークゲートウェイ (`low_latency_gateway/`)
このモジュールの目的はスループットではなく、リアルタイムデータ取り込みのためのレイテンシ最小化です。ベアメタルなソケットプログラミングで Binance の WebSocket API に接続し、カーネルレベルの調整（`TCP_NODELAY`）により、市場データをマイクロ秒レベルで受信・処理できるようにしています。このモジュールにはスループットのベンチマークは適用されません。

### ⚙️ 極限エンジニアリングの詳細
このプロジェクトは、現代的な高水準抽象を捨て、低レベルの POSIX プリミティブと厳格なメモリレイアウトを採用しています。

#### 1. ゼロアロケーションのハードコードI/Oパイプライン (`data_ingestion/`)
* **ディスクバイパスストリーミング:** `.zip` ファイルはディスクに展開しません。`unzip -p` を `xargs` で並列化し、C++ バイナリの `stdin` に直接パイプして、SSD 書き込み制限を回避します。
* **小数部ポインタ演算:** `convert.cpp` は標準的な文字列から浮動小数点への変換を捨てています。ASCII 数字を前方1パスで解析し、小数部インデックス（`fraction_count - 8`）を手動で数えて、価格を整数（Satoshis/0.01 USDT）として直列化します。
* **高密度パッキング:** データは `__attribute__((packed))` を用いた25バイト構造体に強制格納され、コンパイラのパディングを排除することで L1/L2 キャッシュヒット率を最大化しています。

#### 2. Out-of-Core CUDA 計算パイプライン (`cuda_compute_engine/`)
* **L3キャッシュとコア固定:** I/O スレッドと計算スレッドは `pthread_setaffinity_np` により特定 CPU コアへ厳密に固定され、コンテキストスイッチや OS スケジューラによる移動のオーバーヘッドを防ぎます。
* **ロックレスなダブルバッファリング:** CPU 側のストレージ取得と GPU 送信の同期は、純粋に `std::atomic` フラグだけで管理されます。
* **ゼロコピー PCIe DMA:** RAM の確保は `cudaHostAlloc` を使って仮想ページングを回避します。`pread` は pinned host memory に直接書き込み、その後 GPU が `cudaMemcpyAsync` により非同期でデータを取り込み、ディスクレイテンシを完全に隠蔽します。

#### 3. ベアメタルネットワークゲートウェイ (`low_latency_gateway/`)
* **カーネルレベルソケット:** 高水準の WebSocket ライブラリは完全に排除されています。ゲートウェイは生の `<sys/socket.h>` を OpenSSL で包んで使用します。
* **マイクロ秒レイテンシチューニング:** TCP スタックは明示的に調整されています。Nagle アルゴリズムを無効化（`TCP_NODELAY`）し、OS が小さなネットワークパケットをまとめるのを防ぐことで、微小なボラティリティ信号を即時に伝搬させます。
* **生フレームのアンパック:** コードは WebSocket プロトコルヘッダをビット演算で直接解析し、JSON ペイロードの位置を特定します。これにより hot path 上での `std::string` の中間ヒープ確保を回避しています。
