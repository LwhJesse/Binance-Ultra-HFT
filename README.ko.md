## 🇰🇷 한국어 버전

> ⚠️ **엄격한 고지 사항 및 이 프로젝트의 성격**
>
> 1. **이식성 제로:** 이것은 **범용 라이브러리**가 **아닙니다**. 이 코드베이스는 개발에 사용된 시스템의 특정 하드웨어 토폴로지(CPU 캐시 라인, PCIe 버스 대역폭, VRAM 한계)에 강하게 하드코딩되어 있으며 엄격하게 결합되어 있습니다. **다른 PC에서 실행하면 높은 확률로 Segfault가 발생하거나 성능이 심각하게 저하될 수 있습니다.**
> 2. **유지보수 없음:** 이것은 Binance 틱 데이터만을 위해 제작된 완성된 성능 PoC입니다. 유지보수, 업데이트, 지원을 제공할 계획이 없습니다(정말 심심해지지 않는 한).
> 3. **투자 조언 아님:** 이 엔진은 실거래 환경에서의 실행 안정성을 보장하지 않으며, 수익성은 더더욱 보장하지 않습니다. 실제 자금으로 사용하지 마십시오.
>
> **왜 오픈소스로 공개했는가?** C++/CUDA의 극단적인 베어메탈 최적화 기법을 보여주기 위해서입니다. 소스 코드를 연구하고, 네트워크/연산 패러다임을 추출하고, 기술적 구현을 논의하는 것은 환영합니다.

### 모듈 및 성능 사양

이 프로젝트는 서로 다른 성능 목표를 가진 두 개의 독립적인 구성 요소로 이루어져 있습니다.

#### 파트 1: 과거 데이터 배치 처리 (`data_ingestion/` 및 `cuda_compute_engine/`)

이것은 핵심 처리량 엔진이며, 하드웨어의 물리적 한계까지 밀어붙여 대규모 과거 데이터셋을 파싱하고 계산하도록 설계되었습니다. 아래 벤치마크는 **오직 이 모듈에만** 해당합니다.

![NVIDIA Nsight Profiling](README.assets/nsight-profile.png.png)
*(CPU `pread`, PCIe Host-to-Device DMA, GPU 커널 실행이 완벽하게 겹쳐져 있음)*

**대상 하드웨어(이 환경에 맞춰 하드 튜닝됨):**
* **CPU:** Intel Core i7-13650HX (14코어, 20스레드)
* **GPU:** NVIDIA GeForce RTX 4060 Laptop GPU (8GB GDDR6)
* **RAM:** 16GB DDR5 4800MHz (엄격한 물리적 제약으로 인해 Out-of-Core 아키텍처를 강제)
* **Storage:** PCIe 3.0 NVMe SSD

**실행 지표:**
* **데이터셋:** Binance BTC/USDT Spot 과거 체결 데이터 139 GB. 파서는 이 특정 CSV 형식에 하드코딩되어 있으므로, ETH 또는 다른 심볼/데이터 유형에 사용하면 **파싱 실패가 발생할 가능성이 매우 높습니다**.
* **실행 시간:** **약 60초** (실제 스톱워치로 측정).
* **지속 처리량:** **약 2.31 GB/s**.
> *성능 관련 참고: 60초의 안정 실행 버전은 16GB RAM 제약에 맞춰 튜닝되었습니다. 더 공격적인 실험 빌드는 **약 3.23 GB/s**의 최고 처리량(139GB를 43초, 역시 스톱워치 측정)에 도달했지만, 공격적인 메모리 사전 할당 전략 때문에 제한된 16GB 물리 RAM을 초과하여 시스템 프리징을 유발할 위험이 있습니다.*

#### 파트 2: 실시간 네트워크 게이트웨이 (`low_latency_gateway/`)
이 모듈의 목표는 처리량이 아니라 실시간 데이터 수집에서의 지연 최소화입니다. 베어메탈 소켓 프로그래밍을 사용해 Binance WebSocket API에 연결하며, 커널 수준의 튜닝(`TCP_NODELAY`)을 통해 시장 데이터가 마이크로초 단위로 수신 및 처리되도록 합니다. 이 모듈에는 처리량 벤치마크가 적용되지 않습니다.

### ⚙️ 극한 엔지니어링 세부 사항
이 프로젝트는 현대적인 고수준 추상화를 버리고, 원시 POSIX 프리미티브와 엄격한 메모리 레이아웃을 선택합니다.

#### 1. 무할당 하드코딩 I/O 파이프라인 (`data_ingestion/`)
* **디스크 우회 스트리밍:** `.zip` 파일은 디스크에 풀지 않습니다. `unzip -p`를 `xargs`로 병렬화하고 C++ 바이너리의 `stdin`으로 직접 파이프하여 SSD 쓰기 한계를 우회합니다.
* **소수부 포인터 연산:** `convert.cpp`는 표준 문자열-실수 변환을 버렸습니다. ASCII 숫자를 한 번의 전방 패스로 파싱하면서 소수부 인덱스(`fraction_count - 8`)를 수동으로 계산하여 가격을 정수(Satoshis/0.01 USDT)로 직렬화합니다.
* **고밀도 패킹:** 데이터는 `__attribute__((packed))`를 사용한 25바이트 struct에 강제로 담겨 컴파일러 패딩을 제거함으로써 L1/L2 캐시 적중률을 극대화합니다.

#### 2. Out-of-Core CUDA 연산 파이프라인 (`cuda_compute_engine/`)
* **L3 캐시 및 코어 고정:** I/O 스레드와 연산 스레드는 `pthread_setaffinity_np`를 사용해 특정 CPU 코어에 엄격히 고정되어, 컨텍스트 스위칭 오버헤드와 OS 스케줄러의 마이그레이션을 방지합니다.
* **락 없는 더블 버퍼링:** CPU 스토리지 fetch와 GPU 제출 사이의 동기화는 오직 `std::atomic` 플래그만으로 관리됩니다.
* **제로카피 PCIe DMA:** RAM 할당은 `cudaHostAlloc`를 사용해 가상 페이징을 우회합니다. `pread`는 pinned host memory에 직접 기록하고, 이후 GPU가 `cudaMemcpyAsync`를 통해 비동기적으로 가져가므로 디스크 지연이 완전히 가려집니다.

#### 3. 베어메탈 네트워크 게이트웨이 (`low_latency_gateway/`)
* **커널 수준 소켓:** 고수준 WebSocket 라이브러리는 완전히 제거되었습니다. 게이트웨이는 OpenSSL로 감싼 원시 `<sys/socket.h>`를 사용합니다.
* **마이크로초 지연 튜닝:** TCP 스택은 명시적으로 튜닝됩니다. Nagle 알고리즘을 비활성화(`TCP_NODELAY`)하여 OS가 네트워크 패킷을 묶지 못하게 하고, 미세 변동성 신호가 즉시 전달되도록 합니다.
* **원시 프레임 언패킹:** 코드는 비트 연산으로 WebSocket 프로토콜 헤더를 직접 파싱해 JSON payload 위치를 찾으며, hot path에서 `std::string`의 중간 heap 할당을 피합니다.
